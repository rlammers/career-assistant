# Security readiness backlog

Deployment remains blocked while the application completes its public-demo readiness work.

The public work categories are:

1. Define the lifecycle of safe fictional demo data.
2. Validate the selected persistent-storage design.
3. Validate production ingress and proxy behavior.
4. Complete browser-facing security hardening.
5. Complete cloud operational and cost controls.
6. Strengthen build and dependency supply-chain controls.
7. Record explicit acceptance for any remaining public-demo risk.

Exact weaknesses, test observations, thresholds, topology details, and exit criteria are stored in the ignored local file `docs/security-review-private.md`.
